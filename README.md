## Архитектура репозитория (Задание 6)

```text
root/
├── docs/                 # Документация (ТЗ, отчеты по неделям)
│   ├── week1/            # Отчет за 1-ю неделю
│   ├── week2/            # Отчет за 2-ю неделю
```

## Стратегия ветвления (Задание 6)

Для проекта выбрана классическая стратегия **Git Flow**.


| Тип ветки | Префикс в имени | Пример имени | Назначение | От какой ветки создается? | Куда сливается? |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Основная** | `main` | `main` | Стабильная версия продукта | - | - |
| **Разработка** | `develop` | `develop` | Интеграция новых функций | `main` | `main` (при релизе) |
| **Новая функция** | `feature/` | `feature/auth-module` | Разработка новой фичи | `develop` | `develop` |
| **Исправление бага** | `fix/` или `bugfix/` | `fix/login-error` | Срочное исправление ошибки | `main` или `develop` | `develop` / `main` |
| **Релиз** | `release/` | `release/v1.0` | Подготовка к выпуску версии | `develop` | `main` и `develop` |
